# muse-cute

優しく愛らしい、ふんわりした見た目を与える系統。
`muse-interface`（共通の UI 構造設計）の上に、可愛いの見た目トーンを与える。

`muse-playful` が「明るく元気（高彩度・弾む）」なのに対し、
`muse-cute` は「優しく穏やか（淡色・やわらか）」に寄せる点で区別する。

## この系統の狙い / 与える印象

- 可愛い・優しい・癒やし。安心して触れる柔らかさ
- 主張は控えめで、ふんわりした親しみを与える

## 配色・トーン

- ベースはパステル（淡いピンク・水色・クリーム）。低彩度・高明度
- アクセントも淡色寄りで、強い対比は避ける
- テキストは黒でなく、やわらかい茶・グレー寄り

## タイポグラフィ

- 丸ゴシックや手書き風で柔らかさを出す
- 文字サイズの差は控えめに。角の立たない字形を選ぶ
- 多用しすぎず、余白で軽さを保つ

## レイアウト・余白

- 余白は多め。ゆったりと呼吸する間隔
- 要素は小ぶりで、角が丸い。詰め込まない

## 装飾・質感・モーション

- 角丸は大きく統一。影は淡くぼかす（ふわっと浮く程度）
- ドット・小花・ハートなどの控えめなモチーフ
- モーションはゆっくり・やわらかいフェードやふくらみ（過度に弾ませない）

## 避けること（アンチパターン）

- 高コントラスト・原色のぶつけ合い・鋭い角
- 装飾を盛りすぎてうるさくなる
- 業務系で軽すぎて信頼感を損なう用途での濫用

## 出力（hephaestus に渡す見た目指針）

- 配色（パステルのベースと淡いアクセント）の方向
- 角丸の大きさと、淡くぼかした影の強さ
- 丸ゴシック/手書き風の字種と、控えめなモチーフの使いどころ
