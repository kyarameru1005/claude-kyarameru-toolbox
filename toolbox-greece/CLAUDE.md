# CLAUDE.md

## 基本原則

- 日本語で簡潔に応答する。
- 推測より確認を優先する。
- 変更は必要最小限にとどめる。

## 作業範囲と安全

- 変更は `cwd` 配下に限定する。
- `cwd` 外や破壊的操作は事前確認する。
- 秘密情報や実行時データは追加しない。

## Git

- 作業前に Git の状態を確認する。
- 必要に応じて `claude/` で始まる作業ブランチへ分けてから変更する。

## オーケストレーションと再開性

- 役割分担を明示したいときは zeus または `/zeus-orchestrate` を入口にする。
- 長い作業や複数エージェント作業では、目的・現在地・完了済み・未完了・次の一手を短く残す。
- 複雑な作業では zeus がチェックリストで完了・進行中・未着手を管理する。

### タスク台帳（moira）

会話はトークン枯渇で要約・断絶しうるため、長時間作業ではディスク上の台帳を真実にする。

- 短い作業や単発作業は、インラインのチェックリストで足りる。moira は使わない。
- 長時間・複数エージェント・トークン枯渇が懸念される作業では、`moira`（`apps/moira` のCLI）で
  `.ai/moira.json` に台帳を作り、目的・現在地・次の一手・決定ログ・タスク状態を残す。
- 中断後の再開時は、最初に `moira show` を読み、完了済みは Git と突き合わせてから次の一手に進む。
- `.ai/` は実行時データなのでコミットしない（各プロジェクトのローカル状態）。
