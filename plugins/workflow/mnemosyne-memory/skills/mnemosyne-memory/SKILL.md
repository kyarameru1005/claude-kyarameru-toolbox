# mnemosyne-memory

CLAUDE.md（メモリ）を編集した後に、グローバルと照らし合わせて重複を削り、文脈を最小に保つために使う。

## 前提（CLAUDE.md の読み込みモデル）

- 該当する CLAUDE.md は全て **加算** で context に入る。内容の重複排除はされない。
- 同じ内容を複数ファイルに書くと、その分だけトークンを二重に消費する。
- 原則：共通ルールはグローバル（`~/.claude/CLAUDE.md`）に1つ、プロジェクト固有だけを各リポジトリの `CLAUDE.md` に置く。
- 競合する記述は、より具体的（プロジェクト）が優先される。

## 重視すること

- 対象 CLAUDE.md とグローバルを必ず読み比べてから判断する（推測で消さない）。
- 削除候補は「グローバルに同義がある＝重複/包含」されるものに限る。
- 競合（同じ主題で逆の指示）は削らず、意図を確認する。プロジェクトが優先される点を明示する。
- 破壊的に消す前に、削除差分を提示して確認する。グローバル側は依頼が無ければ触らない。
- プロジェクト固有・例外・このリポジトリだけの規約は残す。

## 手順

1. 対象を特定する。明示が無ければ直近で編集した `CLAUDE.md`、なければプロジェクト直下の `./CLAUDE.md` を対象にする。
2. グローバル `~/.claude/CLAUDE.md` を読む。存在すれば cwd から上位階層の `CLAUDE.md` も対象に含める。
3. 対象の各項目を3分類する。
   - 重複/包含：グローバルに同義あり → 削除候補。
   - 競合：同主題で内容が異なる → 残す＋確認（プロジェクト優先）。
   - 固有：このプロジェクトだけのもの → 残す。
4. 削除候補をまとめ、適用後にどれだけ削れるか（行数・概算トークン）を示す。
5. 確認の上で、対象ファイルから重複だけを最小差分で削除する。
6. 「共通化すべきなのにグローバルに無い」項目は、グローバルへ移す提案をする（移動であって複製を増やさない）。
7. 複数リポジトリで共有したい断片は、コピーではなく `@パス` インポートでの一本化を提案する。

## やらないこと

- グローバル `CLAUDE.md` の無断編集（依頼があるときだけ）。
- 競合項目の独断削除。
- 読み比べをしない推測削除。

## 出力

- 対象ファイルとグローバルのパス
- 重複/包含（削除候補）の一覧
- 競合の一覧（確認事項）
- プロジェクト固有として残す項目
- 適用する最小差分と、削減できる行数・概算トークン
- グローバルへ寄せる提案や `@import` による一本化提案があればそれ
