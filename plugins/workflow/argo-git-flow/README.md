# argo-git-flow

Git と GitHub CLI を使い、作業前のブランチ確認、ブランチ作成や切り替え、ステージング、コミット、プッシュ、gh による PR 作成、PR 作成後の CI 確認と自己修正ループを安全に進めるときに使う。破壊的な Git 操作、意図しないファイルの混入、秘密情報の誤追加を避けながら、CI 通過まで含めて Git 作業を完了させたい場合に向いている。

## 動作確認シナリオ

- シナリオ: 変更をコミットして PR を作り、CI が通るまで見届けたい。
- 入力例: 「この変更をブランチを切ってコミットし、PR まで作って CI が通るのを確認して」
- 期待する発火: `argo-git-flow` スキル。
- 期待する挙動の要点:
  - `git status` 確認 → `claude/` ブランチ発行 → パス指定ステージング → コミット → push → `gh pr create` の順で進める。
  - PR 作成後は `gh pr checks --watch` で CI を監視し、失敗時はログ取得 → ローカル再現 → 修正 → 再 push を最大 5 回まで繰り返す。
  - 5 回で通らない場合は停止し、原因分析と経緯を報告してユーザーの判断を仰ぐ。
- 逸脱の例（NG）:
  - PR 作成や push で作業を終え、CI の失敗を放置する。
  - CI を通すためにテストの削除・skip 追加・CI 設定の無効化を無断で行う。
  - force push で CI をやり直す。
