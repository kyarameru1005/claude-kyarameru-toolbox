# olympus-orchestrate

すぐ実装に入るより、役割分担と進め方を先に決めることが重要なときに使う。

## 対応サブエージェント

このスキルは Manager の `zeus` サブエージェントを入口にする。Zeus が必要な支援を判断し、調査は
`hermes`、設計は `daedalus`、実装は `hephaestus`、レビューは `athena`、検証は `themis`、
セキュリティは `ares`、文書は `apollo`、記録は `chronos`、耐障害性実験は `khaos` へ委譲する。
判断・委譲・並列化・台帳運用の詳細は zeus サブエージェントを真実とする。

## 重視すること

- 目的・前提・完了条件を先に短くそろえる
- 必要な役割だけを選び、使う／使わない理由を示す
- 独立作業は並列、順序依存（調査→設計→実装→レビュー→検証）は直列に保つ
- 誰へ何を依頼し、その結果をどう使ったかを最後に報告する

## 手順

1. 目的・前提・完了条件を整理する。
2. ファイル変更を伴う作業では、最初に Git 状態を確認する。
3. 必要な役割を選び、Zeus に判断と委譲を任せる。
4. 複雑な作業はチェックリスト（基本は `moira` 台帳）で進捗を管理する。
5. 完了前に検証結果と未確認事項をまとめる。

## 出力

- 判断と理由
- 進行計画（並列グループがあれば同時発火と直列順を明示）
- 必要な場合だけタスクチェックリスト
- 人間が実装前に理解すべきこと
- 他エージェントを使った場合だけ委譲記録
