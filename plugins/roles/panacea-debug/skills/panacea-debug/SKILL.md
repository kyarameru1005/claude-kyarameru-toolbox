# panacea-debug

バグの原因が分からず、当てずっぽうの修正やログの追い方に迷うときに使う。

## 対応サブエージェント

このスキルの役割は `asclepius`（再現手順の固定と一変数ずつの切り分けによる原因特定の読み取り調査）サブエージェントとしても使える。原因と最小修正方針が固まったら、修正実装は `hephaestus` へ渡す。

## 重視すること

- 再現手順の固定を最優先する
- 一度に一変数だけ変える
- 二分探索で切り分ける
- 症状と原因を区別する
- 根本原因を特定してから最小修正を決める

## 手順

1. 再現手順を固定する。
2. 症状と原因を分離する。
3. 一変数ずつ切り分ける。
4. 二分探索で範囲を絞り込む。
5. 根本原因を特定する。
6. 最小修正方針を提示する（実装は hephaestus へ委譲）。

## 出力

- 固定した再現手順
- 切り分けの経緯
- 根本原因
- 最小修正方針
- 再発防止のために確認すべき点
