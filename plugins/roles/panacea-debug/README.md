# panacea-debug

バグ・不具合・再現する異常挙動を、再現手順の固定→一変数ずつの切り分け→二分探索→根本原因特定の順で直すときに使う。当てずっぽうの修正やログの追い方に迷うときに向いている。

## 動作確認シナリオ

- シナリオ: 再現するバグの原因が分からず、当てずっぽうな修正になりがちな状況。
- 入力例: 「このテストが時々失敗する。原因を切り分けて」
- 期待する発火: `panacea-debug` スキル、`asclepius` エージェント。
- 期待する挙動の要点:
  - 再現手順を先に固定する。
  - 一度に一変数だけ変えて二分探索で切り分ける。
  - 根本原因を特定し最小修正方針を提示する（実装はしない）。
- 逸脱の例（NG）:
  - 再現を固定せず当てずっぽうに修正を始める。
  - `asclepius` がファイルを変更する。
