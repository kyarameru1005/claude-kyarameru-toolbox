# poseidon-integrate

API・他システム・webhook・MCP など外部サービスとの接続と相互運用の方針を検討する外部連携専用サブエージェント Poseidon を導入する。接続方式の選定、認証方式の整理、障害時の切り分け方針を実装前に決めたいときに使う。

Poseidon は方針提示にとどめ、外部への送信・登録（MCP サーバーの登録・設定変更を含む）は実行しない。疎通確認は開発者が実行すべきコマンドを手順として提示する。

## 動作確認シナリオ

- シナリオ: 外部 API との連携機能を作る前に、接続方式と認証の扱いを決めたい。
- 入力例: 「決済 API と webhook で連携したい。実装前に接続方式と認証情報の扱いを整理して」
- 期待する発火: `poseidon` エージェント。
- 期待する挙動の要点:
  - 接続方式・認証方式・データ受け渡し形式を整理し、権限と取得情報の範囲を最小限にする方針を示す。
  - 秘密情報は環境変数や secret 参照で渡す設計にし、値そのものは書かない。
  - 障害時の切り分け手順（開発者が実行する確認コマンド）を提示する。
- 逸脱の例（NG）:
  - curl 等で外部エンドポイントへ実際に通信する。
  - Git 操作・PR・CI を実行する（Git ワークフロー用スキル、例: argo-git-flow の領分）。
  - バージョン一括更新やリリース作業を行う（リリース管理用スキルの領分）。
  - システム全体のアーキテクチャ設計に踏み込む（daedalus の領分）。
