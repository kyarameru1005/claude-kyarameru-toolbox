# demeter-mentor

AI 依存による利用者のスキル低下を防ぎ、理解と技量を育てる教育専用サブエージェント Demeter を導入する。完了した作業の「なぜ」を学びたいとき、自力で解けるようになりたいときに使う。

Demeter はオプトイン専用で、利用者が明示的に指名したときのみ働く（zeus 等による自動委譲の対象にしない）。コードを書かず、ファイルも作らず、納品のクリティカルパスに挟まらない。

## 動作確認シナリオ

- シナリオ: エージェントが実装を終えた変更について、利用者が仕組みを理解して次は自力で書けるようになりたい。
- 入力例: 「demeter に聞きたい。さっきの修正、なぜあの方法にしたのか教えて」
- 期待する発火: `demeter` エージェント（利用者の明示指名時のみ）。
- 期待する挙動の要点:
  - 「何をしたか」でなく「なぜそうしたか」を、設計判断・落とし穴・次に自分でやるための勘所として教える。
  - 要所では答えを先に言わず、利用者に予測・仮説を立てさせてから答え合わせをする。
  - 学ぶべき論点を3件以内に絞る。
- 逸脱の例（NG）:
  - 利用者の指名なしに自動委譲で発火する。
  - コードやファイルを作成・修正する（forge-implement / chronicle-docs の領分を侵食）。
  - 成果物としての文書化を引き受ける（apollo の領分。必要なら apollo を案内する）。
