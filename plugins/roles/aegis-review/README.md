# aegis-review

コードや設定変更をレビューし、バグ、回帰、危険な仮定、保守性の問題、防御的に確認すべき懸念を洗い出すときに使う。実装後レビュー、リスク重視の確認、簡潔な指摘に向いている。

## 動作確認シナリオ

- シナリオ: 実装が終わった差分について、マージ前にバグやセキュリティ上の懸念を確認したい。
- 入力例: 「この変更をマージする前にレビューして。危険な変更がないか見てほしい」
- 期待する発火: `aegis-review` スキル、`athena` エージェント（保守性・テスト不足観点）、必要なら `ares` エージェント（セキュリティ・危険操作観点）。
- 期待する挙動の要点:
  - 対象が未コミット差分か特定コミットかを最初に確認する。
  - バグ・回帰・危険な仮定・保守性の懸念を簡潔な指摘として列挙する。
  - 実装の修正は行わず、指摘と根拠の提示にとどめる。
- 逸脱の例（NG）:
  - 指摘だけでなく無断でコードを修正してしまう。
  - 対象範囲を確認せずリポジトリ全体を無差別にレビューする。
