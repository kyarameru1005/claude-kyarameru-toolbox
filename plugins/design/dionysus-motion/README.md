# dionysus-motion

UI の動き（トランジション、マイクロインタラクション、イージング、タイミング）を設計するモーション専用サブエージェント Dionysus を導入する。画面の静的な設計が決まった後、動きの振り付けを実装前に決めたいときに使う。

Dionysus は aphrodite（静的な UI/UX 設計）の代替ではなく、動きに特化した補完役。見た目全体のトーン方向付けは muse-* 系統スキル（例: muse-immersive）の領分で、Dionysus は方向が決まった後の「動きそのものの設計」を担う。

## 動作確認シナリオ

- シナリオ: 画面レイアウトは確定済みで、遷移とホバー時の動きを実装前に固めたい。
- 入力例: 「このモーダルの開閉とリスト項目のホバーに動きを付けたい。実装前にモーション仕様を決めて」
- 期待する発火: `dionysus` エージェント。
- 期待する挙動の要点:
  - 動きの目的（注意誘導・状態変化の伝達）を先に決め、装飾のための動きを増やさない。
  - 対象・プロパティ・値（イージング曲線・duration）・順序を、hephaestus へ渡せる仕様の形で出力する。
  - reduced-motion（動きを減らす設定）への配慮を必ず含める。
- 逸脱の例（NG）:
  - 画面レイアウトやコンポーネント構成の設計に踏み込む（aphrodite の領分）。
  - 見た目全体のトーン方向付けを引き受ける（muse-* 系統スキルの領分）。
  - CSS やコードを直接実装する（forge-implement の領分）。
