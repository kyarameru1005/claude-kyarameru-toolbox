# agents

このディレクトリは、サブエージェント定義を置く場所です。
各 `*.md` は 1 つのサブエージェント設定を表し、frontmatter に `name`, `description`, `tools`, `model` を、本文にシステムプロンプト（役割・行動指針）を持ちます。

## 形式

```markdown
---
name: agent-name
description: いつこのエージェントを使うかの説明
tools: Read, Grep, Glob
model: sonnet
---

ここにシステムプロンプト（役割、行動方針、出力形式）を書く。
```

## 基本方針

- 1つのエージェントに複数の責務を詰め込まない。
- 調査、設計、実装、レビュー、検証の責務を分ける。
- まず必要最小限の役割だけを作り、必要になったら増やす。

## 典型的な役割

- 判断役: 何を自分で処理し、何を委譲するかを決める。
- 調査役: 関連ファイル、依存関係、影響範囲を整理する。
- 設計役: 方針、責務分離、拡張性を整理する。
- 実装役: 設計に沿って変更を加える。
- 評価役: レビュー、テスト、リスク確認を行う。

## 注意点

- 書き込み権限（`Edit`, `Write`）を持つエージェントは最小限にする。
- 読み取り専用エージェントは調査や評価に向いている。
- 役割名は自由だが、責務が名前から分かる状態を保つ。
