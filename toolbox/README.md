# toolbox

この toolbox は、初期状態へ戻すための Claude Code 設定の原本です。
役割ごとにサブエージェントを分け、必要なものだけを追加しながら育てる前提で使います。

## サブエージェント

- `agents/` には役割別のサブエージェント定義（`*.md`）を置く。
- 1つのエージェントに複数の責務を詰め込まず、調査、設計、実装、レビューを分ける。
- まず最小の役割だけ作り、必要になった段階で追加する。

## オーケストレーション

- メインの Claude は全体判断と依頼の分配を担当する。
- 調査が必要なら探索用エージェント、実装が必要なら実装用エージェントに委譲する。
- 大きな作業は「調査 → 設計 → 実装 → 検証」の順で分ける。
- 並列化は独立した作業だけに限定し、同じファイルを複数エージェントで同時に触らせない。
- 結果を統合する前に、差分とテスト結果をメイン側で確認する。

## ディレクトリ

- `CLAUDE.md`: 全体の行動原則
- `settings.json`: Claude Code の基本設定
- `agents/`: サブエージェント定義（frontmatter 付き `*.md`）
- `commands/`: スラッシュコマンド定義
- `skills/`, `hooks/`, `plugins/`, `mcp/`, `memory/`: 必要になった時点で使う拡張用ディレクトリ
